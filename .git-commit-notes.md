# Git Setup & Commit Instructions

Git installation via winget completed, but may require a system restart or PATH update for PowerShell to recognize it.

## Manual Commit Steps

Once Git is available in your terminal, run:

```powershell
Set-Location 'c:\GabrielData\portfolio'
git init
git config user.name "Gabriel"
git config user.email "your-email@example.com"
git add .
git commit -m "Initial portfolio commit: HTML, CSS, JS with data import, assets, and helper scripts"
```

## Files to Commit

- `index.html` — main page with import UI
- `styles.css` — responsive styles
- `script.js` — data import, rendering, drag/drop
- `data/projects.json` — sample projects
- `assets/thumb1.svg`, `thumb2.svg`, `thumb3.svg` — placeholder images
- `serve.ps1` — start server script
- `stop-serve.ps1` — stop server script
- `README.md` — documentation
- `.gitignore` — ignore patterns

## Troubleshooting

If `git` is still not recognized:
1. Restart your PowerShell/terminal session
2. Or add Git to PATH manually: `$env:PATH += ';C:\Program Files\Git\cmd'`
3. Or verify Git installed: `Test-Path 'C:\Program Files\Git\bin\git.exe'`
